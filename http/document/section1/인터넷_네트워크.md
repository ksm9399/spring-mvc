### `IP(인터넷_프로토콜)`

> `인터넷 프로토콜의 역할`
>  - 지정한 IP주소에 데이터 전달
>  - 패킷이라는 통신 단위로 데이터 전달

> `IP 프로토콜의 한계`
> 1. 비연결성
>    - 패킷을 받을 대상이 없거나, 서비스 불능 상태여도 패킷 전송
> 2. 비신뢰성
>    - 중간에 패킷이 사라지면?
>    - 패킷이 순서대로 오지 않으면?
> 3. 프로그램 구분
>    - 같은 IP를 사용하는 서버에서 통신하는 애플리케이션이 둘 이상이라면?

> `패킷(packet)이란`
> - 컴퓨터 네트워크가 전달하는 데이터의 형식화된 블록

---

### `TCP와 UDP`

> `인터넷 프로토콜의 스택 4계층` <br>
> 애플리케이션 계층 - HTTP, FTP <br>
> 전송 계층 - TCP, UDP <br>
> 인터넷 계층 - IP <br>
> 네트워크 인터페이스 계층

> `TCP 특징(Transmission Control Protocol - 전송 제어 프로토콜)`
> - 연결 지향 - TCP 3 way handshake(가상연결)
> - 데이터 전달 보증
> - 순서 보장
> - 신회할 수 있느 프로토콜
> - 현재는 대부분 TCP사용

> `TCP 3 way handshake - connect과정`
> 1. SYN - 접속 요청
> 2. SYN+ACK - 접속 요청 + 요청 수락
> 3. ACK - ACK와 함께 데이터 전송 가능

> `UDP 특징(User Datagram Protocol - 사용자 데이터그램 프로토콜)`
> - 하얀 도화지에 비유(기능이 거의 없음)
> - 연결지향 X - TCP 3 way handshake X
> - 데이터 전달 보증 X
> - 순서 보장 X
> - 데이터 전달 및 순서가 보장되지 않지만, 단순하고 빠름
> - IP와 거의 같다. PORT정보, 체크섬 정도만 추가
> - 애플리케이션에서 추가 작업 필요

### `PORT`

> `PORT 정보`
> - 0 ~ 65535 할당 가능
> - 0 ~ 1023: 잘 알려진 포트, 사용하지 않는 것이 좋음
> - FTP - 20, 21
> - TELNET - 23
> - HTTP - 80
> - HTTPS - 443

`IP = 아파트` <br>
`PORT = 아파트의 ~동 ~호`

### `DNS`

> `DNS(Domain Name System - 도메인 네임 시스템)`
> - 전화번호부와 같음
> - 도메인 명을 IP주소로 변환

`IP는 변경될 수 있다`
